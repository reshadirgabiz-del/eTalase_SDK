TypeScript SDK · v1.0

The storefront API client for eTalase.

A framework-agnostic TypeScript module that lets any custom website connect to the eTalase storefront API — browse products, manage a cart, run checkout, apply promo codes, and track orders, with a single public store key.

Production ready TypeScript first ESM + CommonJS No framework lock-in Zero secret keys

One-line install

Drop into any web project — Next.js, Vite, plain HTML — with the same API surface.

Public-key only

Browser-safe. Domain authorization is enforced by the backend, never your code.

Full checkout flow

Catalog, cart, delivery rates, promos, orders, and proof-of-payment uploads.

Persistent cart

Opt-in localStorage sync, scoped per store, with subscribe-based updates.

Requirements

  • TypeScript 4.7+ recommended (any version works at runtime).
  • Browser with native fetch and ES2020 support.
  • Node.js 18+ if used in server-side contexts.

Initialize the client

The EtalaseClient wires together every API namespace and the cart manager. Pass your public store key — the SDK sends it as x-etalase-store-key on every request.

typescript
import { EtalaseClient } from 'etalase-module';

export const etalase = new EtalaseClient({
  storeKey: 'store_public_xxx',
  apiUrl: 'https://api.e-talase.com', // optional
  persist: true,                       // optional, enables cart localStorage
});

Factory form

If you prefer composition over new, the factory returns the same instance:

typescript
import { createEtalaseClient } from 'etalase-module';

export const etalase = createEtalaseClient({
  storeKey: 'store_public_xxx',
});
Production default
apiUrl is optional. When omitted, the SDK uses https://api.e-talase.com.

Concepts

A short tour of the moving parts.

The client

EtalaseClient is a thin orchestrator. It owns a single EtalaseApiClient (handles HTTP) and instantiates one namespace per API domain. Everything you need hangs off the client instance.

PropertyPurpose
client.storePublic store profile and storefront settings.
client.productsCatalog browsing, paginated lists, photos only.
client.cartLocal in-memory cart with optional persistence.
client.deliveryCourier rate estimates.
client.promoPromo code validation.
client.ordersOrder creation, tracking, payment proof.
client.orderLinksPublic order link resolution.
client.apiLow-level HTTP for advanced cases.

Public vs secret keys

The SDK accepts only public store keys (store_public_*). Public keys identify the storefront and rely on the backend's per-domain authorization. Never paste a backend secret into client code.

Cart lives in the browser

CartManager never calls the backend. It maintains an in-memory list of items, fires events on mutation, and (with persist: true) mirrors itself to localStorage under etalase-cart-<storeKey>.

Domain security

The SDK does not verify merchant domains itself. It sends the public store key, the browser attaches the current Origin, and the backend decides whether the combination is allowed.

The SDK sends

x-etalase-store-key on every request, plus a normal browser fetch from your storefront domain.

The browser sends

The Origin header automatically. Application code must not set it manually.

The backend validates

The public key, the origin, and whether that origin is verified for the store.

The SDK never uses

Secret keys, wildcard CORS, manual Origin, or mode: "no-cors".

Add your origin to the dashboard
Before testing on a new domain (including http://localhost:5173), add the exact origin in your eTalase dashboard. Unauthorized origins fail with the unauthorized-domain error — see Error handling.

Store

Read the merchant storefront profile and public settings.

GET store.getInformation() → { info, settings }

One-shot helper that combines profile and settings into a single object.

GET store.getInfo() /stores/:id/public

Public store profile — store name, store photo URL.

GET store.getSettings() /settings/public

Storefront settings — currency, payment methods, delivery defaults, social links.

typescript
const store = await etalase.store.getInformation();

console.log({
  name: store.info?.storeName ?? store.settings?.storeName,
  photoUrl: store.info?.storePhotoUrl ?? store.settings?.logoUrl,
  currency: store.settings?.currency,
  payments: {
    bankTransfer: store.settings?.bankTransferEnabled,
    midtrans: store.settings?.midtransEnabled,
  },
});

Products

Browse the merchant catalog. All methods return strongly-typed responses.

GET products.list({ page?, limit? }) → PaginatedResponse<Product>

Paginated product list with full product data (price, stock, images, variants).

GET products.get(id) → Product

Single product by ID.

GET products.listPhotos({ page?, limit? }) → PaginatedResponse<ProductPhotoSummary>

Lightweight list of product IDs, names, and photos only — ideal for gallery views.

GET products.getPhotos(id) → ProductPhotoSummary

Photos for a single product.

typescript
const { data, total } = await etalase.products.list({ page: 1, limit: 20 });
const product = await etalase.products.get(data[0].id);

Cart

A local cart with an event-emitter API and optional localStorage persistence. The cart never calls the backend by itself; it produces the payload that checkout, delivery, and promo APIs consume.

MethodSignatureDescription
localaddItem(product, quantity?, variant?)Add a product or variant. Increments quantity if already present.
localremoveItem(key)Remove by productId or productId:variantId.
localupdateQuantity(key, quantity)Set new quantity. Values ≤ 0 remove the line.
localclear()Empty the cart.
localgetItems()Return a snapshot of current cart items.
localgetTotal()Sum using discountedPrice when available.
localgetItemCount()Total quantity across all lines.
localgetTotalWeightGrams()Total cart weight (defaults to 500g per item if missing).
localtoCheckoutItems()Map to the shape consumed by checkout and promo APIs.
localsubscribe(cb)Run cb after every mutation.
localunsubscribe(cb)Detach a listener.

Subscribing to changes

typescript
function render() {
  const count = etalase.cart.getItemCount();
  document.querySelector('#cartBadge')!.textContent = String(count);
}

etalase.cart.subscribe(render);
render();
Persistence key. When persist: true, the cart is stored at etalase-cart-<storeKey>. Each store has its own bucket.

Delivery

POST delivery.estimate(payload) /delivery/estimate · → DeliveryOption[]

Returns the courier options and prices the merchant has enabled.

typescript
const options = await etalase.delivery.estimate({
  destinationAddress: 'Jl. Sudirman No. 1, Jakarta',
  totalWeightGrams: etalase.cart.getTotalWeightGrams(),
  locationId: 'optional_location_id',
});

Promo codes

POST promo.validate(payload) /promo-codes/validate · → PromoValidationResult

Validate one or more codes against the current cart and selected delivery price.

typescript
const promo = await etalase.promo.validate({
  codes: ['HEMAT10', 'WELCOME'],
  items: etalase.cart.toCheckoutItems(),
  deliveryPrice: 15000,
});

console.log(promo.valid, promo.invalid, promo.finalTotal);

Orders

POST orders.create(payload) /orders · → Order

Creates an order. The SDK injects the configured public store key into the request body for backend compatibility — you do not need to pass storeId.

GET orders.track(orderId, phone) /orders/track · → PublicOrderTracking

Public status lookup. The phone number is the recipient phone on the order.

POST orders.submitProof(orderId, file, phone) /orders/:id/proof · → { proofUrl }

Upload bank transfer payment proof. file is a browser File or Blob.

Create an order

typescript
const order = await etalase.orders.create({
  items: etalase.cart.toCheckoutItems(),
  address: {
    recipientName: 'Budi Santoso',
    phone: '08123456789',
    street: 'Jl. Melati No. 12',
    city: 'Jakarta Selatan',
    province: 'DKI Jakarta',
    postalCode: '12190',
  },
  deliveryOption: options[0],
  promoCodes: ['HEMAT10'],
  paymentMethod: 'bank_transfer',
});

Track an order

typescript
const tracking = await etalase.orders.track('order_id', '08123456789');
console.log(tracking.status, tracking.trackingNumber);

Upload transfer proof

typescript
const input = document.querySelector<HTMLInputElement>('#proof');
const file = input?.files?.[0];

if (file) {
  const result = await etalase.orders.submitProof('order_id', file, '08123456789');
  console.log(result.proofUrl);
}

Order links

GET orderLinks.getPublic(linkId) /order-links/:id/public · → OrderLinkPublic

Resolve a merchant-created public order link. Useful for "buy these items" sharable URLs.

typescript
const link = await etalase.orderLinks.getPublic('link_123');

for (const item of link.items) {
  if (item.product) {
    etalase.cart.addItem(item.product as any, item.quantity);
  }
}

Recipes

Drop-in examples for common storefront flows.

End-to-end checkout

typescript
// 1. Add to cart
const product = await etalase.products.get('prod_abc123');
const variant = product.variants?.find((v) => v.name === 'Large');
etalase.cart.addItem(product, 2, variant);

// 2. Estimate delivery
const options = await etalase.delivery.estimate({
  destinationAddress: 'Jl. Melati No. 12, Jakarta Selatan',
  totalWeightGrams: etalase.cart.getTotalWeightGrams(),
});

// 3. (Optional) validate a promo
const promo = await etalase.promo.validate({
  codes: ['HEMAT10'],
  items: etalase.cart.toCheckoutItems(),
  deliveryPrice: options[0].price,
});

// 4. Create the order
const order = await etalase.orders.create({
  items: etalase.cart.toCheckoutItems(),
  address: {
    recipientName: 'Budi Santoso',
    phone: '08123456789',
    street: 'Jl. Melati No. 12',
    city: 'Jakarta Selatan',
    province: 'DKI Jakarta',
    postalCode: '12190',
  },
  deliveryOption: options[0],
  promoCodes: promo.valid.map((p) => p.code),
  paymentMethod: 'bank_transfer',
});

// 5. Show next steps
if (order.paymentMethod === 'midtrans' && order.midtransRedirectUrl) {
  window.location.href = order.midtransRedirectUrl;
} else {
  console.log('Transfer to:', order.bankDetails);
}

Lightweight product gallery

typescript
const { data } = await etalase.products.listPhotos({ page: 1, limit: 12 });

const html = data
  .map((p) => `<img src="${p.photos[0]?.imageUrl}" alt="${p.productName}" />`)
  .join('');

document.querySelector('#gallery')!.innerHTML = html;

Validate promo before checkout

typescript
const promo = await etalase.promo.validate({
  codes: ['HEMAT10', 'WELCOME'],
  items: etalase.cart.toCheckoutItems(),
  deliveryPrice: 15000,
});

console.log({
  validCodes: promo.valid.map((p) => p.code),
  invalidCodes: promo.invalid,
  finalTotal: promo.finalTotal,

Resolve a shared order link

typescript
const link = await etalase.orderLinks.getPublic('link_123');

for (const item of link.items) {

    etalase.cart.addItem(item.product as any, item.quantity);

Framework usage

The package is framework-agnostic. There are no React components or hooks; integrate it with the patterns your stack already uses.

React / Next.js

etalase.ts
import { createEtalaseClient } from 'etalase-module';

export const etalase = createEtalaseClient({
  storeKey: process.env.NEXT_PUBLIC_ETALASE_STORE_KEY!,
  persist: typeof window !== 'undefined',
});
useCart.ts
import { useSyncExternalStore } from 'react';
import { etalase } from './etalase';

export function useCart() {
  return useSyncExternalStore(
    (cb) => {
      etalase.cart.subscribe(cb);
      return () => etalase.cart.unsubscribe(cb);
    },
    () => etalase.cart.getItems(),
    () => [],
  );
}

Vue 3

composables/useCart.ts
import { ref, onMounted, onUnmounted } from 'vue';
import { etalase } from './etalase';

export function useCart() {
  const items = ref(etalase.cart.getItems());
  const sync = () => { items.value = etalase.cart.getItems(); };

  onMounted(() => etalase.cart.subscribe(sync));
  onUnmounted(() => etalase.cart.unsubscribe(sync));

  return { items };
}

Plain JavaScript / CDN

html
<script type="module">
  import { EtalaseClient } from 'https://esm.sh/etalase-module';

  const etalase = new EtalaseClient({ storeKey: 'store_public_xxx' });
  const products = await etalase.products.list();
  console.log(products.total);
</script>

Error handling

The SDK throws three distinct kinds of errors. Use the class to discriminate cleanly.

ErrorThrown when
Error (generic)Missing or invalid configuration (e.g. no storeKey).
EtalaseApiErrorBackend returns a non-2xx response. status and message are populated.
EtalaseNetworkErrorBrowser fails to reach the backend, typically because of a CORS block.
typescript
import {
  EtalaseApiError,
  EtalaseNetworkError,
  UNAUTHORIZED_DOMAIN_MESSAGE,
} from 'etalase-module';

try {
  await etalase.products.list();
} catch (error) {
  if (error instanceof EtalaseApiError) {
    console.log(error.status, error.message);
  }

  if (error instanceof EtalaseNetworkError) {
    console.log('Network or CORS failure:', error.message);
  }
}
Missing store key
eTalase SDK requires a public store key. Pass storeKey to ETalaseProvider.
Unauthorized domain
This domain is not authorized for this eTalase store. Add and verify this domain in your eTalase dashboard.

Shopee integration

The Shopee module is a server-side SDK subset for syncing marketplace orders into eTalase. It signs Shopee Open Platform v2 requests, reads order details, and prepares logistics calls for shipping labels and tracking.

Backend only

Do not run this module in browser storefront code. Shopee partnerKey, accessToken, and refreshToken values are secrets and must stay in your backend or scheduled worker.

Import and initialize

Use the dedicated subpath so backend jobs can depend only on the Shopee subset. Store credentials in environment variables or your backend secret manager.

typescript
import { createShopeeClient } from 'etalase-module/shopee';

const shopee = createShopeeClient({
  partnerId: process.env.SHOPEE_PARTNER_ID!,
  partnerKey: process.env.SHOPEE_PARTNER_KEY!,
  shopId: process.env.SHOPEE_SHOP_ID!,
  accessToken: process.env.SHOPEE_ACCESS_TOKEN!,
  environment: 'production',
});

Connect a shop

Send the merchant to Shopee's authorization URL, then exchange the callback code for tokens. Persist the returned token pair per store/shop.

typescript
const authUrl = await shopee.createAuthorizationUrl({
  redirectUrl: 'https://api.e-talase.com/integrations/shopee/callback',
  state: storeId,
});

const tokens = await shopee.getTokenByCode({
  code: callbackCode,
  shopId: callbackShopId,
});

await saveShopeeConnection(storeId, {
  shopId: tokens.shop_id,
  accessToken: tokens.access_token,
  refreshToken: tokens.refresh_token,
  expiresIn: tokens.expire_in,
});

Refresh access tokens

Shopee access tokens expire. Refresh them before scheduled sync jobs or when Shopee returns an expired-token error.

typescript
const nextTokens = await shopee.refreshAccessToken({
  refreshToken: connection.refreshToken,
  shopId: connection.shopId,
});

await updateShopeeConnection(storeId, {
  accessToken: nextTokens.access_token,
  refreshToken: nextTokens.refresh_token,
  expiresIn: nextTokens.expire_in,
});

Sync orders

Use listSyncOrders for the normal scheduled sync path. It pages through get_order_list, fetches details with get_order_detail, and returns a normalized shape with the raw Shopee payload attached.

typescript
const orders = await shopee.listSyncOrders(
  {
    timeRangeField: 'update_time',
    timeFrom: new Date(Date.now() - 60 * 60 * 1000),
    timeTo: new Date(),
    pageSize: 50,
  },
  {
    shopId: connection.shopId,
    accessToken: connection.accessToken,
  },
);

for (const order of orders) {
  await upsertMarketplaceOrder({
    storeId,
    source: order.source,
    externalOrderId: order.externalOrderId,
    status: order.status,
    recipient: order.recipient,
    shipping: order.shipping,
    items: order.items,
    totalAmount: order.totalAmount,
    raw: order.raw,
  });
}

Shipping label and tracking

Read the shipping parameters before calling shipOrder. After shipment is arranged, create and poll the shipping document, then download the label. Use getTrackingInfo for tracking events.

typescript
const auth = {
  shopId: connection.shopId,
  accessToken: connection.accessToken,
};

const shipping = await shopee.getShippingParameter(
  {
    orderSn: order.externalOrderId,
    packageNumber: order.shipping.packageNumber,
  },
  auth,
);

await shopee.shipOrder(
  {
    orderSn: order.externalOrderId,
    packageNumber: order.shipping.packageNumber,
    pickup: shipping.pickup,
    dropoff: shipping.dropoff,
  },
  auth,
);

const documentParams = await shopee.getShippingDocumentParameter(
  {
    orderSn: order.externalOrderId,
    packageNumber: order.shipping.packageNumber,
  },
  auth,
);

await shopee.createShippingDocument(
  {
    orderSn: order.externalOrderId,
    packageNumber: order.shipping.packageNumber,
    shippingDocumentType: documentParams.suggest_shipping_document_type,
  },
  auth,
);

const label = await shopee.downloadShippingDocument(
  {
    orderList: [
      {
        orderSn: order.externalOrderId,
        packageNumber: order.shipping.packageNumber,
      },
    ],
    shippingDocumentType: documentParams.suggest_shipping_document_type,
  },
  auth,
);

const tracking = await shopee.getTrackingInfo(
  {
    orderSn: order.externalOrderId,
    packageNumber: order.shipping.packageNumber,
  },
  auth,
);

Operational notes

  • Persist Shopee tokens per eTalase store and Shopee shop.
  • Use update_time for incremental sync jobs and keep a successful-sync watermark.
  • Use externalOrderId as the idempotency key when upserting marketplace orders.
  • Store raw Shopee payloads for reconciliation, but avoid exposing them directly to storefront clients.
  • Handle ShopeeApiError by logging errorCode and requestId.

Backend requirements

The SDK is ready for merchant-hosted domains, but the backend must enforce the security model. If you operate the eTalase backend, ensure each of the following:

  • Accept the public store key from x-etalase-store-key.
  • Read the browser-provided Origin header.
  • Look up the store by its public store key.
  • Allow the request only when the origin is verified for that store.
  • Return CORS headers for the exact allowed origin — never *.
  • Reject unverified domains with a readable 401 or 403 response.
  • Never require backend secret keys in browser SDK usage.

Testing checklist

Manual verification before publishing or going live with a new merchant origin.

  1. Run npm run build and confirm the bundle output exists in dist/.
  2. Load the SDK from the main eTalase app; confirm requests include x-etalase-store-key.
  3. Load from the builder preview after allowlisting that preview origin.
  4. Load from http://localhost:5173 (or your dev port) after allowlisting the exact origin.
  5. Load from the merchant's approved custom domain; confirm requests succeed.
  6. Load from an unverified domain; confirm the SDK surfaces the unauthorized-domain message.

For AI agents

If an AI assistant integrates, modifies, or reasons about this SDK, the canonical contract document is ai-agent.md. It defines the initialization contract, request shape, security model, and what AI agents must not change.

Key invariants
  • Public initialization field is storeKey — never reintroduce storeId.
  • All backend calls must go through EtalaseApiClient.
  • Do not introduce React as a dependency unless the package intentionally becomes a React SDK.
  • Do not use wildcard CORS, manual Origin, or mode: "no-cors".

FAQ

Can I use this on the server (SSR)?

Yes for read-only calls. Server runtimes must provide fetch (Node 18+, modern edge runtimes). Set persist: false — there's no localStorage on the server. For SSR carts, keep cart state on the client.

Does the SDK ship React components?

No. It exports classes, factories, and TypeScript types. ETalaseProvider is a plain class wrapper, not a JSX provider.

Where do I get a public store key?

From your eTalase merchant dashboard. Keys are scoped per store and prefixed store_public_.

Why does my localhost request fail?

The exact origin — protocol, host, and port — must be added to the store's allowed origins in the dashboard. http://localhost:5173 and http://localhost:3000 are different origins to the backend.

How do I clear the persisted cart?

Call etalase.cart.clear(). The persisted entry under etalase-cart-<storeKey> is rewritten immediately.
